single ranked table with everything an LLM needs to talk about the
homologs; non-UniProt subjects (NCBI RefSeq / GenBank) are flagged in
`notes` rather than silently dropped.

hand. The capture above shows the partial-failure model: KEGG's
`NotFoundError` is recorded inline in step 3, the other three sources
complete, and the rank-merge proceeds across whatever did succeed.

**Why use this over chaining manually?** The tool is sequence-aware
ortholog reconciliation in one call: it fetches AT1G01010's protein
sequence from UniProt, runs Gramene compara homology and NCBI BLAST in
parallel against that sequence, then joins by a normalized locus key
(stripping Gramene's `SPECIES_` prefix and BLAST's accession version
suffix). The output is one ranked list with `n_sources`, `mean_identity`,
and per-source provenance (`gramene_hit` + `blast_hit` preserved on each
row), eliminating the "I have Gramene's compara-based orthologs and
BLAST's sequence-based homologs, which ones agree" merge step that
otherwise has to live in client code.
