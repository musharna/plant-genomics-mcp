`synthesis.py` imports 9 of the 10 live backend modules (`ensembl_plants`, `uniprot`, `europe_pmc`, `quickgo`, `blast`, `gramene`, `kegg`, `string_db`, `atted`) and composes them. `phytozome` is not used by any v0.8 orchestrator — broadening organism coverage is the v0.9 leverage point. No new HTTP, no new caches.

### Execution model: concurrent gather per orchestrator

Each orchestrator calls its independent backends via `asyncio.gather(*coros, return_exceptions=True)` for parallel execution against the shared `httpx.AsyncClient`. Sequential dependencies (e.g., `analyze_locus_synth` must resolve `ensembl_plants.lookup_locus` before `xrefs`/`uniprot` calls that need the Ensembl gene ID) are handled by running the gather in two phases:

1. **Phase 1 — root resolution.** `await` the single root call (e.g., `lookup_locus`). If this fails, downstream phase-2 rows are `status="skipped"`.
2. **Phase 2 — concurrent fan-out.** `asyncio.gather(...)` across all dependent backends in parallel.

### Caching

No composite cache. Each backend's existing `TTLCache` continues to handle repeated calls. Composition cost is `O(n)` Python over already-cached envelopes — not worth a second cache layer.

## Tool Contracts

All four return a `SynthesisEnvelope` (Pydantic, `ConfigDict(extra="forbid")`). MCP `outputSchema` derived via `model_json_schema()` as elsewhere.

### Shared types

```python
class StepRow(BaseModel):
    model_config = ConfigDict(extra="forbid")
    step: int                                # 1-indexed in execution order
    tool: str                                # backend tool name (e.g., "ensembl_plants_lookup_locus")
    status: Literal["ok", "error", "skipped"]
    elapsed_s: float
    result: dict | list | None = None        # backend-shaped payload when status="ok"
    error: str | None = None                 # "[ExceptionClass] message" when status="error" or skip reason when status="skipped"

class SynthesisEnvelope(BaseModel):
    model_config = ConfigDict(extra="forbid")
    tool: str                                # e.g., "analyze_locus_synth"
    input: dict                              # echoed input args
    started_at: str                          # ISO 8601 UTC
    elapsed_s: float                         # total wall time
    steps: list[StepRow]
    result: dict | None                      # composed result, None if root step failed
```

`error` strings use the existing `[ExceptionClass] message` format from `errors.PlantGenomicsError.__str__` — clients can already parse this.

### 4.1 `analyze_locus_synth(locus: str, species: str | None = None) -> SynthesisEnvelope`

Mirrors the `analyze_locus` prompt (5-tool chain). Default species = `"arabidopsis_thaliana"` (matches prompt default).

- **Phase 1.** `ensembl_plants.lookup_locus(species, locus)` → ensembl gene record.
- **Phase 2 (gather).** `ensembl_plants.get_xrefs`, `uniprot.lookup_locus`, `europe_pmc.search_locus`, `quickgo.lookup_annotations` — all parallel.

`result` shape when phase-1 succeeds:

```python
{
  "ensembl_record": {...},           # from phase 1
  "xrefs": [...] | None,             # None if step errored
  "uniprot_record": {...} | None,
  "literature": {...} | None,
  "go_annotations": [...] | None,
  "reconciled": {
    "canonical_gene_name": str | None,         # from ensembl_record.external_name, fallback uniprot.gene_name
    "best_uniprot_accession": str | None,      # from uniprot_record.primary_accession
    "conflict_flags": list[str]                # e.g., ["gene_name_mismatch", "uniprot_xref_disagreement"]
  }
}
```

`result` is `None` when phase 1 errors (root step recorded in `steps[0]`).

### 4.2 `find_homologs_synth(sequence: str, program: Literal["blastp", "blastn", "tblastn", "blastx"] = "blastp", top_n: int = 10) -> SynthesisEnvelope`

Mirrors the `find_homologs` prompt. `top_n` capped at 50 (BatchEnvelope convention).

- **Phase 1.** `blast.search(sequence, program)` → BLAST hits (truncated to `top_n`).
- **Phase 2.** Collect UniProt-shaped subject accessions from the top-N hits, dedupe, single call to `uniprot.batch_lookup_accession(accessions)` (the existing batch helper from P1.11). One rolled-up StepRow regardless of hit count.

`result.ranked_hits` is a list of `{rank, blast_hit, uniprot_record | None}` (uniprot record matched by accession). Hits without a parseable UniProt accession set `uniprot_record = None` and add a single `"non_uniprot_subject"` flag to a top-level `result.notes` list.

### 4.3 `biological_context_synth(locus: str, species: str | None = None, top_n: int = 10) -> SynthesisEnvelope`

Mirrors the `biological_context` prompt. Default species = `"arabidopsis_thaliana"`. `top_n` capped at 50.

- **Phase 1.** `uniprot.lookup_locus(locus, species)` → UniProt accession needed by STRING.
- **Phase 2 (gather).** `gramene.homologs(locus, species, top_n)`, `kegg.pathways(locus, species)`, `string_db.partners(uniprot_acc, top_n)`, `atted.coexpression(locus, species, top_n)` — all parallel.

`result` shape:

```python
{
  "uniprot_accession": str,
  "homologs": [...] | None,
  "pathways": [...] | None,
  "string_partners": [...] | None,
  "atted_coexpression": [...] | None,
  "consensus_partners": [                       # ranked merge of string_partners + atted_coexpression
    {"target_locus": str, "n_sources": int, "combined_score": float, "sources": list[str]},
    ...
  ]
}
```

`consensus_partners` ranking: `combined_score = mean(normalized_scores)` where STRING `combined_score` is already 0–1, ATTED MR (mutual rank) is normalized via `1 / (1 + log1p(MR))`. Sort by `(n_sources desc, combined_score desc)`, top-`top_n`.

If phase 1 errors, all phase-2 rows are `status="skipped"` with `error="phase-1 UniProt resolution failed; downstream calls skipped for atomic envelope semantics"`. Gramene and KEGG could in principle still run (they key on locus, not accession), but skipping all of phase 2 keeps the envelope atomic: every `biological_context_synth` either has a complete coordinated result set or signals the root failure unambiguously. If a future use case wants partial-on-phase-1-fail, that's a v0.9+ change.

### 4.4 `consensus_homologs(locus: str, species: str | None = None, top_n: int = 10) -> SynthesisEnvelope`

Pure synthesis — no equivalent prompt. Default species = `"arabidopsis_thaliana"`. `top_n` capped at 50.

- **Phase 1.** `uniprot.lookup_locus(locus, species)` → sequence for BLAST query.
- **Phase 2 (gather).** `gramene.homologs(locus, species, 50)`, `blast.search(uniprot_record.sequence, "blastp", top_n=50)` — parallel. (Both pull 50 raw hits to give consensus more material.)

**Dedup.** Group hits by `(target_species_lower, target_gene_or_locus_lower)`. Gramene `target_gene` is taken directly. BLAST subject IDs are heterogeneous (NCBI, UniProt, plant-specific accessions) — Task 5 in the implementation plan begins by reading `blast.py` to confirm whether a parser helper exists; if not, the task adds a small `_parse_blast_subject_for_consensus()` helper inside `synthesis.py` that handles the accession formats actually present in plant BLAST results (NCBI `>sp|`, `>tr|`, `>ref|`, plant TAIR/Phytozome locus-shaped IDs). The helper's coverage is asserted by unit tests with real-shape fixtures.

**Scoring.** For each group:

- `n_sources = len({src for src in group})` (1 or 2)
- `mean_identity = mean(identity for hit in group)` where:
  - Gramene contributes `identity = 1.0` (Gramene homology calls do not return alignment identity; treat as full-confidence orthology call)
  - BLAST contributes `identity = pident / 100`
- `score = n_sources * mean_identity`

**Output.** Sort by `(score desc, target_species)`, return top-`top_n`. `result.consensus = [{target_species, target_gene, n_sources, sources: list[str], mean_identity, score, gramene_hit: {...} | None, blast_hit: {...} | None}, ...]`.

If phase 1 errors, both phase-2 rows are `status="skipped"`. If only one of Gramene/BLAST succeeds, consensus runs over that single source — `n_sources` will be 1 for every group, so the rank degenerates to identity-sorted single-source output. This is a graceful degradation, not a special case.

## Error Handling

- Each orchestrator wraps phase-2 in `asyncio.gather(..., return_exceptions=True)`.
- Returned `BaseException` → `StepRow(status="error", error=f"[{type(e).__name__}] {e}")`.
- Returned value → `StepRow(status="ok", result=value)`.
- Top-level `result` is composed from `status="ok"` rows only; `None`-valued fields signal absent data.
- **No new exception classes.** Backend exceptions propagate with their existing `[ExceptionClass] message` formatting from `errors.PlantGenomicsError.__str__`.
- **No fail-loud override:** even when all phase-2 backends fail, the envelope returns `result` composed from whatever succeeded. This is the user-approved partial-envelope semantics.
