## Executive synthesis — what stands between v0.9 and v1.0

Three audit lenses converged on a consistent picture. **Code health is solid** — no shell injection, no SSRF, typed errors, clean caches, deep synthesis abstraction that competitors don't have. **Two leak surfaces dominate the must-fix list:**

1. **Hosted HTTP endpoint is open by default** — no auth, no rate limit, no body-size cap, unbounded `Retry-After`. Cannot ship 1.0 to a public registry like this.
2. **v0.9 multi-organism contract has two broken seams** — `string_interactions` + `batch_string_interactions` silently drop `organism=`, and 7 of 12 Phytozome IDs are unverified because `verify_organisms.py` crashes at import.

Below them, four IMPORTANT findings shape the API-stability story for 1.0, and ten POLISH items shape the registry/marketing surface.

## Recommended ordering — three waves to 1.0

The dependency graph makes the wave structure obvious. Wave A is small mechanical fixes that unlock everything else. Wave B is the auth/abuse hardening that gates public-registry submission. Wave C is the polish/distribution surface that converts "1.0 release" into "1.0 launch."

### Wave A — Multi-organism contract repair (~half day, all S)

Closes BLOCKERs #3, #4, #5 + IMPORTANT #10, #11. All mechanical.

- **A1.** Fix `scripts/verify_organisms.py:22` to source from `organisms.ORGANISMS` instead of removed `phytozome.KNOWN_ORGANISMS`.
- **A2.** Run `verify_organisms.py` against the 7 unverified organisms; populate real `phytozome_int` values for those with stable Phytozome IDs. For any that don't resolve, document explicitly in the coverage matrix.
- **A3.** Add `organism` to `string_interactions` and `batch_string_interactions` inputSchema; forward to `string_db.lookup_partners` in `server.py:1145-1150` and `batch.py:250-261`. Update tool description (drop "Arabidopsis locus" wording).
- **A4.** Strengthen `test_tool_schemas_use_organism_param` to assert `organism` IS present on the explicit set of multi-organism tools (allowlist-driven).
- **A5.** Add `@live_only` synthesis test for `analyze_locus_synth(Os01g0100100, organism="oryza_sativa")` to `tests/test_synthesis.py`.
- **Commit boundary:** one commit per concern (`fix(organisms): …`, `feat(string): organism= contract`, `test(synthesis): live rice probe`). Tag nothing yet.

### Wave A.5 — BAR/Araport scoping probe (~0.5 day investigation, then implementation if greenlit)

Closes IMPORTANT #27. **Decision gate before Wave B locks in 1.0 scope.**

User pushback 2026-05-23: TAIR is a load-bearing resource in plant genomics; the stub redirect is too conservative for 1.0. Real TAIR API requires subscription keys (per 2026-05-21 live probe), which breaks the "intentionally keyless" invariant. **BAR/Araport (Bio-Analytic Resource for Plant Biology, U. Toronto, https://bar.utoronto.ca/)** hosts the academic-mirror of TAIR data (eFP browser expression, ePlant, co-expression networks, AthaMap promoter elements, AraNet PPI) and is reported to expose public REST endpoints.

- **A5a.** Live-probe BAR endpoints (start with eFP expression and ePlant integrated views): document base URL, auth posture (confirm keyless), rate-limit behavior, response shapes, stability signals (changelog, version header).
- **A5b.** Map BAR's distinct value-add vs. existing 9 backends — what does it provide that Ensembl Plants / UniProt / KEGG / QuickGO / Europe PMC don't? Is it arabidopsis-only or broader?
- **A5c.** Decision branch:
  - **GREENLIGHT** → scope `bar.py` backend module (1-2 tools: `bar_expression`, `bar_co_expression`?) for inclusion in Wave B. Slot 1-2 days of implementation effort before tag.
  - **DEFER** → if API is unstable, undocumented, or value-add overlaps existing backends, keep TAIR/PlantCyc as structured redirects for 1.0 and revisit in 1.1. Document the probe finding in README's TAIR section so users see the rationale.
- **Output:** short scoping memo at `docs/superpowers/audits/2026-05-23-bar-api-probe.md` capturing endpoints tested, decision, scope.

### Wave B — HTTP-transport security hardening (~1 day)

Closes BLOCKERs #1, #2 + IMPORTANTs #6, #7, #8, #9. Required before any public registry resubmission.

- **B1.** Bearer-token Starlette middleware on `/mcp` (let `/healthz` pass). Token in `PLANT_GENOMICS_MCP_HTTP_TOKEN`; startup aborts if missing or <32 chars. README + compose docs updated.
- **B2.** Cap `Retry-After` at 60s across all 9 backend `_get`/`_post` helpers. Single shared constant `MAX_RETRY_AFTER = 60.0`. Mechanical, 9 sites.
- **B3.** Add Starlette body-size middleware (1 MB cap). Add `"maxLength": 100000` to `sequence` field in `blast_sequence` + `find_homologs_synth` inputSchemas.
- **B4.** Process-level `asyncio.Semaphore(2)` around `blast.blast_sequence`. Set `PLANT_GENOMICS_MCP_NCBI_EMAIL` to real operator email in compose `environment:` block.
- **B5.** `CORSMiddleware(allow_origins=[], allow_methods=["POST", "GET"])` on the Starlette app.
- **B6.** Extract `_LOCUS_RE` to `src/plant_genomics_mcp/validators.py`; apply in `ensembl_plants.lookup_locus`, `ensembl_plants.lookup_xrefs`, `gramene.lookup_homologs`, `kegg.lookup_pathways` before URL construction.
- **B7.** Add integration test: HTTP request without bearer token → 401. With token → 200. With body >1 MB → 413.
- **Commit boundary:** one commit per defense layer.

### Wave C — API-stability + polish (~half day)

Closes IMPORTANTs #12, #13 + POLISH items #14, #15, #17, #18, #19, #20, #21, #23, #24.

- **C1.** Change `StepRow.elapsed_s` to `float | None` in `models.py:467`; update `_gather_phase2` in `synthesis.py:144-158` to pass `None` for phase-2 rows (honest non-attribution). `SynthesisEnvelope.elapsed_s` (top-level total) unchanged. Update docstring to: "Wall time for this step alone. `None` when measured inside a parallel gather (see phase-2 in `synthesis.py`)." Regenerate JSON schema; update tests asserting on field shape. (Option D — locks 1.0 contract honestly; per-step instrumentation deferrable to 1.1 without breaking the API surface.)
- **C2.** README hosted-endpoint framing → "Personal demo endpoint maintained by @mjarnold — best-effort uptime, no SLA, may move or disappear. For production use, self-host (see `deploy/`)." Pair with bearer-token from B1 — demo token distributed out-of-band. (Decision locked 2026-05-23.)
- **C3.** Drop `version` from `/healthz` payload (return `{"status": "ok"}`).
- **C4.** Add BLAST entry to `pgmcp://backends/status` and `pgmcp://cache/stats`.
- **C5.** Add `organism` argument to `biological_context` prompt (matching `analyze_locus`); document KEGG+ATTED arabidopsis-only constraint in tool description.
- **C6.** Move `synthesis.py:460` `re` import to top.
- **C7.** Add retry to `batch_ensembl_plants_lookup_locus` POST OR document the gap in tool description.
- **C8.** Reconcile coverage-matrix column header with README description.
- **C9.** Surface CI / coverage / license badges in README header.
- **C10.** Commit lock file (`uv.lock` or `requirements.txt --exact`) for reproducible Docker builds.
- **Tag v1.0.0** after Wave C closes and full suite is green.

### Wave D — Post-1.0 launch (~1-2 days, optional pre-tag)

- **D1.** PyPI publish (`twine upload`); `pyproject.toml` already declares `[project.scripts]`. `uvx plant-genomics-mcp` validates one-liner install.
- **D2.** Refresh registry listings: modelcontextprotocol.io directory, PulseMCP, Glama. Update copy to lead with multi-organism + synthesis story.
- **D3.** Walkthrough proof transcripts under `examples/` showing cross-organism orthology probe end-to-end. Demonstrates synthesis-tool value over single-backend wrappers.
- **D4.** Refactor: extract shared `_http.py` for retry pattern (POLISH #22). Backports the `Retry-After` cap centrally.

```
A1 (verify script fix)
  └─> A2 (populate phytozome_int)

A5 (rice synthesis live)

B4 (BLAST semaphore)
B5 (CORS)
B6 (locus regex) ── ↑
B7 (auth integration test) ── depends on B1

C1-C8 ─────────────────────} all parallel; C9 / C10 cosmetic

A5a (BAR live-probe) ──> A5b (value-add map) ──> A5c (decision)
  └─> if GREENLIGHT: bar.py backend implementation (slots into Wave B)

Wave A → Wave A.5 (probe + decision) → Wave B → Wave C → tag v1.0 → Wave D
```

The waves are loosely sequenced — Wave A is mechanical so it goes first to clear deck. Wave B is the heavyweight tier and shouldn't be interrupted by polish noise. Wave C lands the API-contract corrections before tag.

## Decisions locked 2026-05-23

1. **Hosted endpoint framing → personal demo.** README copy: "Personal demo endpoint maintained by @mjarnold — best-effort uptime, no SLA, may move or disappear. For production use, self-host (see `deploy/`)." Pairs with bearer-token from B1. Implemented in C2.
2. **TAIR/PlantCyc 1.0 story → probe BAR/Araport.** User pushback: stub redirect is too conservative for a load-bearing plant-genomics resource. BAR (academic mirror, keyless) is the candidate. Wave A.5 probes API stability + value-add; A5c decision branch determines whether `bar.py` ships before 1.0 tag or defers to 1.1.
3. **PyPI publish gate → v1.0.0 is final publish.** No release-candidate step. PyPI lands in Wave D, immediately after the v1.0.0 tag. (Per user 2026-05-23: "publish pypi once 1.0 and fully ready".)
4. **`StepRow.elapsed_s` → Option D (nullable, phase-2 returns None).** Promoted from POLISH to IMPORTANT #12. Locks 1.0 API contract honestly without ruling out v1.1 per-step instrumentation. Implemented in C1.

## What this does NOT cover

- Performance benchmarking (latency per tool, throughput under load) — not in audit scope; competitor research recommends future work.
- Vulnerability scanning of pinned deps via `pip-audit` / `safety` — recommended as part of CI but not done as part of this audit.
- Threat modeling of the MCP protocol itself (e.g. prompt-injection via tool outputs containing untrusted upstream text) — audited at a surface level; affirmation captured but not exhaustive.
- Documentation of the synthesis-envelope schema (`SynthesisEnvelope.result.{tool}.shape`) — partial; full JSON-schema documentation could be a 1.1 polish item.
